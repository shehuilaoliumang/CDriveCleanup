# 部署指南

## 运行环境

- Windows 10/11 推荐。
- Python 3.7+。
- 项目内已有 `.venv` 时优先使用虚拟环境。
- 建议管理员权限运行，以获得完整系统清理权限。

## 安装依赖

核心功能不依赖第三方库。完整功能建议安装：

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

可选功能对应关系：

- `matplotlib`：磁盘可视化和图表。
- `pystray` + `Pillow`：系统托盘。
- `pyinstaller`：打包为 exe。
- `psutil`：系统性能监控。

## 本地运行

```powershell
.\.venv\Scripts\python.exe main.py
```

或双击：

```text
启动.bat
```

## 打包为 exe

运行：

```powershell
.\.venv\Scripts\python.exe build.py
```

打包成功后会生成：

```text
dist/CDriveCleanup.exe
```

`build/` 和 `dist/` 是构建产物，可以删除后重新生成。

## 发布前检查

1. 确认 `main.py` 能编译通过。
2. 运行自动化测试：

```powershell
.\.venv\Scripts\python.exe run_tests.py
```

3. 确认启动界面能正常显示。
4. 确认右侧滚动条和窗口宽度自适应正常。
5. 确认“刷新智能建议”和“计算可清理空间”不会阻塞界面。

## 可删除文件和目录

以下内容不属于源码：

```text
build/
dist/
__pycache__/
tests/__pycache__/
tmp/
*.spec
```

如果 `tmp/` 中目录权限异常，需要管理员权限清理。

## 常见部署问题

### 双击启动失败

确认 `.venv\Scripts\python.exe` 存在。如果不存在，安装 Python 后重新创建虚拟环境，或让 `启动.bat` 使用系统 Python。

### 打包失败

先安装 PyInstaller：

```powershell
.\.venv\Scripts\python.exe -m pip install pyinstaller
```

如果安全软件拦截 PyInstaller 产物，可将项目目录加入信任列表。

### exe 运行权限不足

右键以管理员身份运行。

### 文档或终端显示乱码

项目文档按 UTF-8 保存。PowerShell 如果显示乱码，可先执行：

```powershell
chcp 65001
```
