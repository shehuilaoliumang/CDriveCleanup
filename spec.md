# 项目规格说明

## 目标

提供一个 Windows 桌面端 C 盘扫描与安全清理工具，帮助用户发现磁盘空间占用、清理可删除缓存、管理大文件和重复文件。

## 用户对象

- Windows 普通用户。
- 需要清理 C 盘空间的用户。
- 需要查看大文件、重复文件和缓存占用的用户。

## 核心需求

### 界面

- 使用 Tkinter 构建桌面界面。
- 支持右侧滚动条。
- 支持内容宽度跟随窗口自适应。
- 支持浅色/深色主题切换。
- 界面启动时不能执行长时间扫描任务。

### 扫描

- 支持选择目录扫描。
- 统计总大小、文件数、目录数。
- 找出大文件。
- 统计扩展名大小。
- 支持增量扫描和中断恢复。

### 清理

- 支持临时文件、浏览器缓存、Windows 更新缓存、Prefetch、缩略图缓存、日志文件等项目。
- 支持白名单。
- 支持清理前备份。
- 支持模拟模式。
- 计算可清理空间应后台执行。

### 重复文件

- 按文件大小初筛。
- 使用哈希识别重复文件。
- 支持选择、移动和删除重复文件。

### 可视化

- 使用 matplotlib 展示目录空间、文件类型和文件分类。
- matplotlib 缺失时显示提示，不影响核心功能。

### 历史和恢复

- 保存扫描历史。
- 保存清理日志。
- 支持从备份目录恢复文件。

### 设置

- 主题记忆。
- 定时清理，支持每 N 天执行一次。
- 系统托盘。
- 右键菜单安装/卸载。
- 依赖检查。
- 打包入口。

## 非功能需求

- 启动界面应快速显示。
- 耗时任务必须后台执行。
- Tkinter 控件只允许主线程更新。
- 文件删除前应提供确认或预览。
- 项目文档使用 UTF-8 编码。

## 已知限制

- 系统目录、回收站、注册表操作可能需要管理员权限。
- 某些文件被占用或 ACL 异常时无法删除。
- 大目录扫描耗时取决于文件数量和磁盘速度。

## 后续优化计划

### 模块拆分

当前 `main.py` 体积较大，后续可以拆分为多个模块，例如：

- `ui_scan.py`：磁盘扫描界面。
- `ui_cleanup.py`：安全清理界面。
- `ui_duplicates.py`：重复文件界面。
- `ui_settings.py`：设置界面。
- `ui_visualization.py`：可视化界面。

模块拆分不会天然影响 exe 打包。只要这些模块通过正常 `import` 被主程序引用，PyInstaller 通常会自动收集。若使用动态导入、插件式导入或额外资源文件，需要在 `build.py` 中补充 `--hidden-import` 或 `--add-data`。

### 定时清理增强

- 增加“执行前确认”选项。
- 增加“仅在系统空闲时执行”选项。
- 增加下一次执行时间预览。
- 支持更多清理分类选择。

### 设置管理

- 支持设置导入/导出。
- 支持一键恢复默认设置。
- 支持记录窗口大小和位置。

### 清理体验

- 为每类清理项显示更详细的文件数量和大小。
- 支持清理任务暂停和取消。
- 增加失败文件列表，方便用户查看哪些文件因权限或占用无法删除。
